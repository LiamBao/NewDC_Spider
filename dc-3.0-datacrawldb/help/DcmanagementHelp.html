SubTask
          用来保存被切分后的任务，包括该任务的描述信息，使用哪个脚本，脚本调用方式，运行间隔，及其执行状况
 long agentId

 long taskId
          关联到SubTaskDefine中的某一个被拆分任务的定义，该任务定义可能会被删除
 java.lang.String subtaskKey
          记录当前任务的key，此属性从拆分脚本中带入
 java.lang.String scriptFile
          记录当前任务执行使用的脚本名称，此属性从任务定义表中带入
 java.lang.String scriptMain
          记录当前任务执行使用的主函数及参数，此属性从任务定义表中带入
 int scrapeCount
 byte status

 long groupId

 int errorCode

 java.lang.String errorMessage
          记录进程的错误信息。
 java.lang.String errorUrl

 byte errorSentFlag

Task
          用来定义新的任务，包括该任务的描述信息，使用哪个脚本，脚本调用方式，运行间隔
 long siteId
          可以用来描述该任务的简短文字，长度为25个汉字，或者50个英文数字符号
 java.lang.String name
          可以用来描述该任务的简短文字，长度为25个汉字，或者50个英文数字符号
 java.lang.String description
          用来描述任务的详细信息，长度为200个汉字，或者400个英文数字符号
 byte splitType
          用来执行该任务的脚本，保存的信息为配置文件中定义的脚本目录下的相对路径，长度为100个英文数字符号
 java.lang.String splitFile
          用来执行该任务的脚本，保存的信息为配置文件中定义的脚本目录下的相对路径，长度为100个英文数字符号
 java.lang.String splitMain
          执行脚本中的主函数及参数名称，长度为300个英文数字符号，例如：execute("http://www.163.com")
 java.lang.String scriptFile
          记录当前任务执行使用的脚本名称，此属性从任务定义表中带入
 java.sql.Timestamp dueTime
          如果超过指定时间，则任务将不再会被任务拆分程序拆分
 boolean dueCheckFlag
          初始时候为true。为false时不发送任务到期提醒邮件
 int dueAlarmBefore
          根据用户指定的天数，在任务到期前给用户发送提醒邮件
 long timeout
          单位：微秒，该值小于1800000（30分钟）的情况下，为未定义超时时间。不然，每个Task开始执行以后，如果超过指定时间，则任务进程停止
 int lastSplitStatus
          标志当前任务最后一次拆分是否错误：0. 正确，1. 出错
 java.lang.String lastSplitMsg
          当last_split_status =1时候，记录的拆分错误信息
 byte enableFlag
          标志当前任务的是否启用：0. 未启用，1. 启用
 byte enableSendFlag
          标志当前任务的是否启用：0. 未启用，1. 启用
 long splitWaitTime
          两次拆分操作的间隔等待时间，单位是毫秒
 int priority
          任务优先级，值越大，表明优先级越高
 java.sql.Timestamp startTime
 int isEnableStartTime
 java.sql.Timestamp lastSplitTime
 java.lang.String lastSplitKey
          拆分批号由随机字符串产生
 int lastSplitCount
          标志当前任务的拆分情况：0. 无需拆分，1. 需要拆分
 long groupId
 long bddId
          0
